카카오엔터 팬덤앱 — 키키(KIKI) 벤치마킹

팬덤앱 전략
⚠️ 핵심 전제 단순히 웹툰 캐릭터를 얹은 팬덤앱이 되면 안 된다. 키키 멤버들이 직접 살아 움직여야 효과가 난다.

키키(KIKI)란?

키키는 여돌 팬덤 타깃 서비스다. 하이브의 위버스(Weverse)가 방탄소년단으로 성장한 것처럼, 키키는 여돌 팬덤을 중심으로 설계된 플랫폼이다.

키키 요소 대응 레퍼런스 핵심 기능
샤라웃(Shout-out) 하이브인더섬 멤버가 직접 유저/캐릭터를 언급
광장 채팅 하이브인더섬 광장 팬들의 실시간 동시 채팅 집결
의상 온타임 출시 - 무대 의상 → 게임 아이템 즉시 반영
마이홈 꾸미기 본디(Bondee) / 싸이월드 꾸미기 구매 + 뽐내기 랭킹

3개 레이어 구조

🌟

멤버 레이어 (핵심)

샤라웃 + 광장 동시 채팅 + 무대 의상 온타임 출시. 멤버가 살아있어야 전체가 작동한다.

🏠

유저 레이어

본디식 마이홈(싸이월드 개념) + 유저간 SNS 인터랙션. 꾸미기 구매 / 방문 / 랭킹 과시 루프.

🎮

게임 레이어

3Match 또는 검증된 캐주얼 게임. 게임 자체가 핵심이 아닌 접착제 역할.

수익 사이클

멤버 샤라웃 → 캐릭터 착장 반영 → 아이템 구매 → 마이홈 과시 / 랭킹 → 재구매 루프
🔑 포지셔닝 조건 키키 멤버들이 "공식 캐릭터처럼" 느껴져야 팬덤이 반응한다. 브랜드 포지셔닝이 전제되어야 수익 사이클이 작동한다.

게임 서사 전략 — 판타지 다마고치 비유

게임 기획 철학
⚠️ 비유임을 명심할 것 "판타지 다마고치"는 장르명이 아니라, 비주얼 서사 설계 전략을 설명하는 은유다.

왜 지금 서사가 중요한가

최근 캐주얼 게임과 H5 게임들은 단순 코어 루프 + 메타 코어(성장)에만 집중한다. 게임이 본질적으로 가져야 할 서사가 없다.

텍스트나 작가 필력으로 사용자를 이끄는 방식은 훅이 약해진 시대에 더 이상 강력하게 작동하지 않는다. 스티키니스를 만들어내기 어려운 환경이다.

해법 — 비주얼 서사

🥚

시작: 무언가 주어진다

게임을 시작하면 플레이어에게 "무언가"가 주어진다. (알, 씨앗, 파편...)

🐉

성장: 눈에 보이게 변한다

게임을 플레이할 때마다 시각적으로 성장한다. 텍스트 설명 없이 비주얼로 전달.

🔮

궁금증: 다음이 보고 싶다

"다음엔 어떻게 되지?"라는 호기심을 자연스럽게 만든다. 직관적 비주얼 서사.

단기·중기·장기 훅 구조

단계 서사 훅 설계 방식
단기 알 → 부화 → 첫 성장 이벤트 트리거형 (플레이하면 즉시 반응)
중기 진화 분기 / 속성 각성 플레이어 선택형 (내 선택이 결과를 만듦)
장기 세계관 개방 / 소셜 확장 다른 유저와의 교류로 확장
🔑 핵심 원칙 작가의 필력에 끌려가지 않는다. 서사를 "이벤트 테이블 + 조건 트리거"로 모듈화하면, 텍스트는 교체 가능한 스킨이 된다. 이것이 AI-Native 서사 설계의 방향이다.

발라트로 동양버전 — 맞고 기반 카드게임

신규 타이틀 기획

발라트로(Balatro)란?

스팀에서 1인 개발로 대성공한 포커 기반 카드 로그라이크 게임이다. 포커 패로 점수를 쌓고, 조커 카드로 시너지를 만드는 단순하지만 깊이 있는 구조가 핵심이다.

1인
개발팀 규모
4개
커버 플랫폼
(H5·모바일·스팀·IP)
맞고
동양버전 베이스

동양버전 전환 설계

발라트로 원본 동양버전 전환
포커 패 (숫자/무늬) 맞고 패 → 전부 캐릭터 이미지로 교체
조커 카드 (특수 강화) 맞고 특수 패 → 발라트로식 변형 적용
솔로 점수 빌딩 미니클립형 대전 방식 베이스
서양 카드 IP 다양한 IP 제휴에 유리한 비주얼 구조

멀티플랫폼 전략

H5 브라우저 → 모바일 앱 → 스팀 PC → IP 제휴 스킨
🔑 IP 제휴 적합성 맞고 이미지를 캐릭터로 완전 교체하는 구조이기 때문에, 어떤 IP를 얹어도 자연스럽다. 웹툰·웹소설 IP를 카드 캐릭터로 변환하면 IP 제휴 게임으로 즉시 전환 가능하다.
